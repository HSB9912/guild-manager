먼저 명확히 — "이중길드"의 정의?
A. 사이트엔 우리 길드(뚠/뚱…)로 등록된 캐릭이, 실제 게임에서는 다른 길드에 가입돼 있는 경우
B. 사이트엔 우리 길드인데, 게임에서는 길드가 없거나(탈퇴 상태)
C. 가입 신청자가 이미 다른 길드 소속 → 신청 시점 감지
기본 가정: A + B 둘 다 잡음 (사이트 길드 ≠ 실제 길드). 우리 길드 목록(뚠/뚱/밤/별/달/꿀/솜)에 안 들어가면 모두 "불일치".

동작: 워커가 멤버 목록 fetch → 넥슨 API 순회 → 불일치 발견 → 디스코드 운영진 채널에 알림 임베드
필요: 워커에 넥슨 API key + 디스코드 웹훅 등록 (보석금 워커처럼)
선후관계: 수동 검사 시스템부터 구축해서 동작 검증 → 자동화는 그 다음. 사용자 요청대로.

설계 결정 사항

1. 메뉴 위치: ① 길드원 메뉴 안 서브탭 / ② 별도 메뉴 "이중길드 감사" / ③ 신청 처리에 새 탭. 추천: ②
2. 검사 범위: 본캐만 / 본캐+부캐 전부 / 선택. 추천: 본캐+부캐 전부 (members 전체)
3. 결과 저장: 휘발성(보고 끝) / DB에 last_guild_check 저장 (재방문 시 결과 유지). 추천: DB 저장 — 자동화 때 활용
4. API 호출 속도: 넥슨 API rate limit 있음 (초당 약 10건 정도). 71명이면 ~10초. 1초당 1건 보수적으로 가면 1분.
5. 길드 없음(탈퇴): 이중길드와 같이 잡지만 색상 분리 (빨강 vs 앰버).
6. 액션: 발견 시 ① 운영진이 수동 탈퇴 처리 ② 알림만 / 자동 탈퇴는 위험. 추천: 알림만 + 운영진 수동
