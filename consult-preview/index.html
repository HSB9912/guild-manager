각 안 특징

A안 — 가장 컴팩트. 한 칸 안에 현재값과 목표값을 위아래로. 카드 높이 거의 안 늘어남.
B안 — "현재 / 목표" 라벨이 명확. 두 줄로 분리되어 알아보기 쉬움. 라벨이 항상 보이는 게 장점.
C안 — "현재 → 교체 후 (+증가량)"까지. 정보가 가장 풍부하지만 카드 높이도 가장 큼. 비교가 직관적.
데이터 출처: 목표 헥사·예산·기간은 이미 입력 받고 있고 goal JSONB에 저장됨. 목표 환산·전투력은 새 입력 필드 추가 필요. 교체 후 헥사는 target_items.hexa_contrib 합산으로 자동 계산 가능.
